### 1.1 XWPF

word文档包括 页眉、页脚、脚注、批注、链接、正文部分，在XWPF中都有与之对应的类：
.docx文件的文档的

根 XWPFDocument
页眉 XWPFHeader
页脚 XWPFFooter
脚注 XWPFFootnote
批注 XWPFComment
链接 XWPFHyperlink
内容主体 

页眉，页脚，脚注，内容主体部分由 段落（XWPFParagraph）、表格（XWPFTable）、图片（XWPFPicture）部分组成.每个类都有相应丰富的方法。了解了这些结构，对于掌握XWPF有很大帮助。每一个XWPF类型，都有与之对应的CT*，个人认为CT*更接近底层操作.

* XWPFParagraph 文档、表格、标题等中的段落

```java
CTP ctp = para.getCTP();//段落可以设置对齐方式，字体，设置标签等操作。
```

* XWPFRun 相同属性设置的文本区域

```java
CTR ctr = run.getCTR(); // XWPFRun是组成XWPFParagraph的元素，对于文本的操作更细致化的在这一个类，区分段落中的XWPFRun非常不容易，因为每个XWPFRun不同体现在很多因素。
```

*  XWPFTable 文档中的表格

```java
CTTbl ct = table.getCTTbl(); // 表格存在行（XWPFTableRow), 每行存在单元格（XWPFTableCell), 每个单元格有段落（XWPFParagraph） 遍历表格：
```

## 微信支付接口调试

### 证书、密钥和签名

* 商户API证书

商户API证书是指由商户申请的，包含商户的商户号、公司名称、**公钥信息**的证书，

API 由微信支付签发（已停用）

APIv3 由CA签发

获取途径： 下载微信支付证书工具->生成证书请求串->提交到商户平台->商户API证书文件

* 商户API私钥

![image-20221223150553995](assets\image-20221223150553995.png)

* 平台证书

*微信支付* 负责申请、包含微信支付平台标识、公钥信息的证书。平台证书周期性更换。商户应定时通过API下载新的证书。

作用：使用公钥进行验签。

获取途径：微信支付平台证书请调用[“获取平台证书接口”](https://pay.weixin.qq.com/wiki/doc/apiv3/wechatpay/wechatpay5_1.shtml) 获取

* API v3密钥

为了保证安全性，微信支付在 *回调通知*和*平台证书*下载接口中，对关键信息进行了`AES-256-GCM`加密。API v3密钥是加密时使用的对称密钥。

获取途径： 商户平台】->【API安全】的页面设置该密钥，请求才能通过微信支付的签名校验。密钥的长度为32个字节。

### 服务器配置要求

a、小程序访问商户服务都是通过HTTPS,开发部署的时候需要HTTPS服务器

b、服务器域名配置

①、每个微信小程序需要事先设置通信域名，小程序只可以跟指定的域名进行网络通信。包括普通 HTTPS 请求（wx.request）、上传文件（wx.uploadFile）、下载文件（wx.downloadFile）和 WebSocket 通信（wx.connectSocket）

②、从基础库 2.4.0 开始，网络接口允许与局域网 IP 通信，但要注意 不允许与本机 IP 通信

注意：

- 域名只支持 https (wx.request、wx.uploadFile、wx.downloadFile) 和 wss (wx.connectSocket) 协议
- 域名不能使用 IP 地址（小程序的局域网 IP 除外）或 localhost
- 可以配置端口，如 https://myserver.com:8080，但是配置后只能向 https://myserver.com:8080/ 发起请求。如果向 https://myserver.com、https://myserver.com:9091 等 URL 请求则会失败
- 如果不配置端口。如 https://myserver.com，那么请求的 URL 中也不能包含端口，甚至是默认的 443 端口也不可以。如果向 https://myserver.com:443 请求则会失败
- 域名必须经过 ICP 备案
- 出于安全考虑，api.weixin.qq.com 不能被配置为服务器域名，相关API也不能在小程序内调用。 开发者应将 AppSecret 保存到后台服务器中，通过服务器使用 [getAccessToken](https://developers.weixin.qq.com/miniprogram/dev/api-backend/open-api/access-token/auth.getAccessToken.html) 接口获取 access_token，并调用相关 API
- 不支持配置父域名，使用子域名
- 可查阅[小程序网络请求](https://developers.weixin.qq.com/miniprogram/dev/framework/ability/network.html)以了解更多信息
